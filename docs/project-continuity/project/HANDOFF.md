## How To Resume

1. Read `AGENTS.md`.
2. Read `docs/project-continuity/project/project-continuity.yaml`.
3. Read purpose, status, next action, user-input gates, handoff notes, decisions, self-improvement policy, and proposed changes.
4. Work only inside approved scope and preserve the repo-specific validation/deploy follow-through rules.
5. End by updating changed continuity files and running `npm run continuity:check`.

## Session-End Updates

Update only facts that changed:

- `STATUS.md` for current state.
- `NEXT.md` for the next recommended action.
- `HANDOFF.md` for resume notes.
- `USER_INPUT.md` for open or resolved gates.
- `DECISIONS/` for durable accepted decisions.
- `PROPOSED_CHANGES.md` for evidence-backed, actionable improvement ideas.
